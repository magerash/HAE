# Custom decision-style inventory

**Purpose:** capture domain-specific decision axes invisible to general personality tests. These are the highest-leverage signals for the twin agent because they map directly to operator behaviors during release planning, scope decisions, code review, and research prioritization.

**Items:** 8.
**Scale:** anchored — each item gives a 1-end and 7-end statement; pick where you sit. 1–7 integer.

## Items

### 1. Risk tolerance

`1` — I'd rather ship a smaller change I'm sure works than a bigger one that might break.
`7` — I'd rather ship a bigger change with real upside even if regression risk is real.

### 2. Scope bias (trim vs. expand)

`1` — When in doubt, cut scope. Smaller PRs win.
`7` — When in doubt, expand scope. If we're already in there, do it right.

### 3. Evidence threshold

`1` — Show me data before I commit. Gut feel is for tiebreakers only.
`7` — I trust my read. Demanding data is often a delay tactic.

### 4. Abstraction tolerance

`1` — Three near-duplicate functions are fine. Don't abstract until the third use.
`7` — If I see two similar functions I want them unified now, before drift.

### 5. Refactor-vs-ship

`1` — Refactor opportunistically only. The job is shipping features.
`7` — Tech debt compounds. Spend a real fraction of every cycle on cleanup.

### 6. Code-review strictness

`1` — Approve and trust. Fast review is more valuable than perfect review.
`7` — Block on every defect I see. The cost of a bug in main is much higher than a slow review.

### 7. Research depth

`1` — Pick a reasonable approach and start. Research is procrastination dressed up.
`7` — Don't start coding until I've surveyed alternatives, read source on contenders, and built a quick spike.

### 8. Parallelism comfort

`1` — One thing at a time. Context switches kill quality.
`7` — Many threads in flight is normal. Parallel exploration finds answers faster.

## Scoring

No reverse-keying — each item is a single 1–7 anchored scale. Output as-is to `profile/custom.json`:

```json
{
  "computed_at": "ISO timestamp",
  "items": {
    "risk_tolerance": 5,
    "scope_bias_expand": 6,
    "evidence_threshold_low": 4,
    "abstraction_tolerance_high": 3,
    "refactor_appetite": 6,
    "review_strictness": 5,
    "research_depth": 6,
    "parallelism_comfort": 7
  },
  "summary": "operator-style descriptor sentence generated from items"
}
```

The `summary` field is generated by the `/hae:profile` skill — a one-sentence persona descriptor synthesized from the 8 ratings, e.g. "high-risk, scope-expanding, research-heavy, comfortable parallelizing — bias toward big bets with deep prep."

## Free-form principles

After scoring, the skill prompts: *"Write 3–8 short principles you operate by, in your own voice. Examples: 'always run codex review before shipping', 'mockups before code', 'ship dev channel before debug channel'."* These go to `profile/principles.md` verbatim. **Highest-signal text** in the entire profile — short, opinionated, operator-authored. Twin agent loads this verbatim into its system prompt.
